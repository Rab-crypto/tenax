Command Reference

Complete reference for all Project Memory commands. All commands use the /project-memory: namespace.

Status & Info

/project-memory:status

Show project memory status and quick summary. Displays total sessions, decisions, patterns, tasks, insights, recent activity, and storage size.

Use when: You want a quick overview of accumulated project knowledge.

/project-memory:stats

Show detailed statistics about project memory including timeline, sessions per month, top topics, and storage breakdown.

Use when: You want analytics on your project history and memory usage patterns.

/project-memory:list

List all stored sessions with metadata including session ID, date, summary, token count, and decision count.

Use when: You want to browse all past sessions and their contents.

Loading Sessions

/project-memory:load-session

Load a specific session by ID into context. Shows full session details including decisions, patterns, files modified, and topics.

/project-memory:load-session <session-id>
ArgumentDescription
session-idSession ID (e.g., "5" or "005")

Example: /project-memory:load-session 003

/project-memory:load-sessions

Load multiple sessions by IDs. If no IDs specified, loads the last 3 sessions. Respects token budget.

/project-memory:load-sessions [IDs] [-b, --budget <tokens>]
ArgumentDescription
IDsSession IDs (comma or space separated)
-b, --budgetToken budget limit

Example: /project-memory:load-sessions 001,002,003

/project-memory:load-recent

Load the N most recent sessions (default: 5). Most recent first. Respects token budget.

/project-memory:load-recent [N] [-b, --budget <tokens>]
ArgumentDescription
NNumber of sessions (default: 5)
-b, --budgetToken budget limit

Example: /project-memory:load-recent 3

/project-memory:load-all

Smart-load all sessions within token budget. Shows preview, loads in reverse chronological order, skips sessions that would exceed budget.

/project-memory:load-all [-b, --budget <tokens>]

Use when: You want comprehensive project history within your token allowance.

/project-memory:fresh

Start a fresh session without loading any history. Memory still records new decisions; you can still search or load on demand.

Use when: You want a clean slate without historical context influencing the conversation.

Recording Knowledge

/project-memory:save

Manually save the current session to project memory. Reviews conversation for decisions, patterns, tasks, and insights.

Use when: You want to checkpoint important knowledge mid-session.

/project-memory:record

Manually record a decision, pattern, task, or insight.

Record a Decision

/project-memory:record decision -t <topic> "<text>" -r "<rationale>"

Example: /project-memory:record decision -t database "Using PostgreSQL" -r "Team expertise"

Record a Pattern

/project-memory:record pattern -n <name> "<description>" -u "<usage>"

Example: /project-memory:record pattern -n error-handler "Wrap async routes in try-catch"

Record a Task

/project-memory:record task "<title>" -d "<description>" -p <priority>

Example: /project-memory:record task "Add unit tests" -p high

Record an Insight

/project-memory:record insight "<text>" -c "<context>"

Example: /project-memory:record insight "Rate limits are per-user not per-app"

Searching

/project-memory:search

Search across all project memory using semantic similarity. Returns ranked results with similarity scores and snippets.

/project-memory:search <query> [-t, --type <type>] [-l, --limit <n>]
ArgumentDescription
queryNatural language search query (required)
-t, --typeFilter by type: decision, pattern, task, insight, session
-l, --limitMaximum results (default: 10)

Examples:

  • /project-memory:search "authentication approach"
  • /project-memory:search "database" -t decision
  • /project-memory:search "error handling" -l 5

Management

/project-memory:settings

View or modify project memory settings including token budget, auto-load behavior, and recording preferences.

See Settings Reference for all available options.

/project-memory:forget

Remove entries from project memory. Confirmation required for bulk deletions.

/project-memory:forget [id] [--type <type>] [--all]
ArgumentDescription
idDelete specific entry by ID
--typeDelete all of type: decisions, patterns, tasks, insights
--allDelete everything (requires confirmation)

Use when: You need to clean up obsolete or incorrect entries.

/project-memory:tag

Add or remove tags from sessions for organization.

/project-memory:tag <session-id> <tag1> [tag2...] [--remove]

Examples:

  • /project-memory:tag 003 auth feature - Add tags
  • /project-memory:tag 003 auth --remove - Remove tag

Export & Backup

/project-memory:export

Export project memory to various formats: Markdown, JSON, Obsidian, or Notion.

/project-memory:export [-f, --format <format>] [-o, --output <dir>] [-s, --sessions]
ArgumentDescription
-f, --formatExport format: markdown (default), json, obsidian, notion
-o, --outputOutput directory
-s, --sessionsInclude full session data

Examples:

  • /project-memory:export - Export to Markdown
  • /project-memory:export -f obsidian -o ./vault - Export to Obsidian vault
/project-memory:backup

Create a backup of all project memory data as a JSON file with base64-encoded contents.

/project-memory:backup [-o, --output <path>]

Use when: You want to safeguard project memory before major changes or share between machines.

/project-memory:restore

Restore project memory from a backup file. Asks for confirmation if memory already exists.

/project-memory:restore <backup-file> [-f, --force]
ArgumentDescription
backup-filePath to backup file (required)
-f, --forceOverwrite without confirmation

Use when: You need to recover from accidental deletions or transfer to a new machine.
